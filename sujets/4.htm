LA MIAGE DE PARIS NANTERRE

Le défi: Ton code est OK ou KO ?

Theme:

développement

Lot:

Un Raspberry Pi par membre de l'équipe gagnante, dans la limite de 300 euros.

La base du défi

Durant la nuit de l'informatique, de nombreux étudiants viennent pour profiter de l'ambiance, sans toutefois se lancer dans un défi. Mettons les à contribution en leur faisant relire votre code de manière ludique !

Vous devez créer l'application web ou mobile définie par le sujet de la nuit de l'informatique. Cependant, vous devrez concevoir cette application de manière à ce que vos camarades puisse valider le code produit durant la nuit de l'informatique via une interface spécifique. Pendant la nuit, vous déposerez une consigne, ou des spécifications et un code associé. Vos camarades pourront dire si le code correspond à la consigne ou si il contient une erreur. Ils auront la possibilité de glisser le code vers la droite pour dire ‘OK, le code réponds à la consigne’, vers la gauche pour dire ‘KO, le code ne correspond pas à la consigne’, vers le haut pour signaler une consigne ou une spécification trop impécise, voire vers le bas pour signaler un code illisible du fait du non respect des normes de codage.

Améliorations

  1. On peut gamifier l’application en ajoutant un score :
    1. Proposer une consigne et un code tiré aléatoirement parmi plusieures version d'une même fonction.
    2. +1 pour une bonne réponse, -1 pour une mauvaise réponse, tableau de classement des joueurs.
    3. Proposer un bonus de points lorsqu’un code est KO en demandant à l’utilisateur de proposer un test unitaire permettant de montrer que le code ne produit pas le bon résultat ou bien une brève explication.
  2. Proposer un filtre pour se concentrer sur un seul langage (PHP, C, C++, JAVA, Python …)
  3. Un tableau de bord pour visualiser facilement les résultats
    1. Pour chaque joueur
    2. Pour chaque consigne
    3. Pour chaque version de code proposée

  4. Toutes vos idées sont les bienvenues, même et surtout les plus farfelues.

Elements attendus

Une application web ou mobile fonctionnelle et originale.

  1. Un lien vers le site en production (ou vers le .apk de l'application mobile)
  2. Les sources du projet (soit par mail, soit un lien vers github)
  3. Un rapport pdf décrivant le projet, l'équipe, la répartition du travail, les choix de conception, les technologies utilisées, le mode d'emploi, et des captures d'écran.

Mode de restitution

Vous devez déposer toutes les informations concernant votre défi, le lien vers le site en production, les sources ou un lien vers les sources ainsi que le rapport via l’interface du site de la nuit de l’informatique. En cas de souci, nous vous conseillons alors d'envoyer un mail au parrain du défi.
